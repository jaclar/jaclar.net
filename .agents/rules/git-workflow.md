# Git & Deployment Workflow Preferences

- **NEVER** run `git commit` or `git push` unless the user explicitly asks for it in their prompt.
- **Conventional Commits**: Always use Conventional Commit format for git commit messages (e.g. `feat(scope): ...`, `fix(scope): ...`, `style(scope): ...`, `refactor(scope): ...`, `docs(scope): ...`).
- Perform all builds, verification, and code edits locally, and present the completed changes to the user first.
